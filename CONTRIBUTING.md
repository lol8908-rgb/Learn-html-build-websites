## 🚀 Getting Started

### Prerequisites
- Basic GitHub knowledge (fork, clone, pull request)
- Text editor (VS Code recommended)
- Git installed on your computer

### Setup

1. **Fork the repository**
   - Click "Fork" in the top right of GitHub
   - This creates your own copy

2. **Clone your fork**
   ```bash
   git clone https://github.com/YOUR-USERNAME/Learn-html-build-websites.git
   cd Learn-html-build-websites
   ```

3. **Create a branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

4. **Make your changes**
   - Edit files in your text editor
   - Follow the style guide (see below)

5. **Test your changes**
   - Read through your changes
   - Make sure formatting is correct
   - Verify code examples work

6. **Commit your changes**
   ```bash
   git add .
   git commit -m "Clear description of what you changed"
   ```

7. **Push to GitHub**
   ```bash
   git push origin feature/your-feature-name
   ```

8. **Create a Pull Request**
   - Go to the original repository
   - Click "New Pull Request"
   - Select your branch
   - Write a clear description
   - Submit!

## 📝 Style Guide

### Markdown Formatting

- **Headings:** Use `#` for main, `##` for sections, `###` for subsections
- **Bold:** `**text**` for important terms
- **Code:** Use backticks for inline code: `` `<tag>` ``
- **Code blocks:** Use triple backticks with language:
  ```html
  <code here>
  ```
- **Lists:** Use `-` for bullets, `1.` for numbered
- **Links:** Use `[text](url)` format

### Lesson Structure

Every lesson should include:
1. **Introduction** — Hook the learner
2. **Explanation** — Clear, simple language
3. **Code Examples** — Working HTML code
4. **Tables** (when helpful) — Organizing info
5. **Try It Yourself** — Hands-on challenge
6. **Tips/Best Practices** — Do's and Don'ts
7. **Common Mistakes** — Prevent confusion
8. **Summary** — Key takeaways
9. **Navigation** — Links to previous/next lesson

### Writing Tips

- ✅ Use simple, clear language
- ✅ Write for beginners (no jargon)
- ✅ Use examples from daily life
- ✅ Keep paragraphs short
- ✅ Use active voice
- ✅ Be encouraging and positive

### ❌ Avoid

- ❌ Complex technical terminology
- ❌ Assuming prior knowledge
- ❌ Very long blocks of text
- ❌ Passive voice
- ❌ Discouraging comments

## 🐛 Reporting Issues

### How to Report a Bug

1. **Check if it's already reported** — Search existing issues
2. **Click "New Issue"**
3. **Use the bug template** if available
4. **Include:**
   - What went wrong
   - What you expected to happen
   - Steps to reproduce
   - Which lesson/file is affected
   - Browser/OS if relevant

### Example Issue Report

```
Title: Lesson 04 code example doesn't work

Description:
The image example in Lesson 04 doesn't display the image.

Steps to reproduce:
1. Copy the code from Lesson 04
2. Save as HTML file
3. Open in browser
4. Image doesn't appear

Expected: Image should display
Actual: No image shown

Browser: Chrome on Windows
```

### How to Suggest Improvements

1. **Click "New Issue"**
2. **Use the enhancement label** (if available)
3. **Describe:**
   - What could be improved
   - Why it would be better
   - Specific location if applicable
   - Any suggested changes

## 🔍 Code Review Process

All contributions go through review:

1. **Maintainer reviews** your pull request
2. **Feedback provided** if changes needed
3. **You can update** your PR based on feedback
4. **Once approved** it gets merged
5. **Thank you!** You're now a contributor

### Review Checklist

We check:
- ✅ Follows style guide
- ✅ Excellent for beginners
- ✅ No errors or typos
- ✅ Code examples work
- ✅ Consistent with other lessons
- ✅ Helpful and clear

## 📚 Translation Guide

Want to translate this course?

### Steps:
1. **Create a new folder** — `lessons-[language-code]`
2. **Copy lesson files** — Rename appropriately
3. **Translate content** — Keep HTML code unchanged
4. **Update README** — Add language to list
5. **Submit as PR** — We'll review and merge
6. **You're credited** — As a translator!

### Language Examples:
- Spanish: `lessons-es/`
- French: `lessons-fr/`
- German: `lessons-de/`
- Japanese: `lessons-ja/`
